## Serenity Gateway Pattern

Serenity requires:
1. A `Handler` struct implementing `EventHandler` trait
2. Handler receives events like `message` when new messages arrive
3. Handler needs access to `AppHandle` to emit Tauri events
4. Client runs in its own async context (separate from commands)

**Key imports needed:**
- `serenity::async_trait`
- `serenity::model::channel::Message`
- `serenity::model::gateway::Ready`
- `serenity::prelude::{Context, EventHandler}`

## Implementation Notes

1. **Handler struct** needs `AppHandle` to emit events
2. **message()** event handler converts `serenity::model::channel::Message` to `Message` model
3. **ready()** event handler logs successful connection
4. **Client initialization** happens in `connect()` alongside HTTP client
5. **Client lifecycle** — run in background task, handle disconnects gracefully
</context>
